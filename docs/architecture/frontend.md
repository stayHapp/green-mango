# 前端规划

本文档记录前端方向，不包含具体代码。

## 页面结构

第一版计划包含：

- 管理端会议列表页
- 管理端会议创建页
- 管理端会议详情页
- 管理端会议编辑页
- 管理端报名字段配置页
- 管理端报名记录页
- 前台会议展示页
- 前台报名提交页

## 组件规划

可逐步抽取以下组件：

- 会议基础信息表单
- 报名字段配置表格
- 报名字段编辑器
- 动态报名表渲染组件
- 报名记录表格
- 字段值展示组件

## API 请求封装

使用 Axios 封装统一请求层，后续至少包含：

- 基础 URL 配置
- 请求错误处理
- 响应数据解包
- 登录态或令牌注入

第一版可以先保持简单，避免过早抽象。

## 路由规划

初步路由分为管理端和前台：

- `/admin/meetings`
- `/admin/meetings/new`
- `/admin/meetings/:id`
- `/admin/meetings/:id/edit`
- `/admin/meetings/:id/fields`
- `/admin/meetings/:id/registrations`
- `/meetings/:id`
- `/meetings/:id/register`

具体路径可在实现前根据交互设计调整。

## 状态管理规划

Pinia 用于保存跨页面共享状态，例如：

- 当前管理员信息
- 当前会议上下文
- 全局加载状态

第一版不应把所有 API 数据都放入全局状态，页面级数据优先在页面内管理。
