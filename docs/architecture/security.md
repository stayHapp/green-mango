# 安全原则

第一版范围较小，但认证、授权和敏感数据边界由后端强制执行。

## 密码与会话

- 管理员和工作人员密码使用 Python 标准库 scrypt 加随机盐计算哈希，数据库不保存明文密码。
- 登录成功签发高熵随机 Bearer token；客户端只在签发时获得原始 token。
- `auth_sessions` 只保存 token 的 SHA-256 摘要、主体、过期时间和撤销时间。
- 默认会话有效期为 12 小时，可通过 `SESSION_EXPIRE_HOURS` 调整。
- 退出登录会撤销当前服务端会话，后续请求立即返回 401。
- 嘉宾按会议 ID、姓名和手机号核验身份，登录后同样使用可过期、可撤销的服务端会话。

## 后台权限

所有管理员和工作人员接口先验证会话主体的角色与启用状态，再验证会议授权关系。前端隐藏按钮不构成权限控制。

会议级资源在不存在和越权时统一返回 404。工作人员只能操作 `staff_meetings` 已授权会议；管理员只能操作 `meeting_admins` 已授权会议。嘉宾只能查看自己所属会议和个人信息。

## 二维码与签到

- 嘉宾二维码只编码随机 token，不包含姓名、手机号等个人信息。
- 后端校验 token、嘉宾启用状态、会议归属、会议结束时间和重复签到。
- 数据库唯一约束保证一个嘉宾在一场会议只有一条签到记录。

## 输入、文件与跨域

- 所有请求体、路径参数和查询参数均由 Pydantic 与业务服务双层校验。
- Excel 导入仅接受 `.xlsx`，限制为 10MB 和 10,000 个数据行，并逐行校验动态字段。
- CORS 默认只允许 Vite 本地开发来源，可通过 `CORS_ORIGINS` 以逗号分隔配置明确来源；不使用通配来源配合凭据。

## 敏感配置

数据库连接、密钥、令牌等敏感配置必须放在环境变量或安全配置系统中。`.env` 不提交到版本库；仓库只提供不含凭据的 `.env.example`。

当前 MVP 未包含短信验证码、分布式登录限流、审计日志和密钥托管。这些属于正式公网部署前的安全加固项，不影响当前三端 API 功能验收。
