# 安全原则

第一版虽然范围较小，但安全边界必须从一开始建立。

## 密码存储

密码不得明文存储。后续如实现管理员登录，必须使用可靠哈希算法保存密码哈希。

## 后台权限

后台接口需要权限控制。前端隐藏按钮不等于安全，后端必须对每个后台接口做权限判断。

## 用户输入校验

所有用户输入都必须在后端校验，包括：

- 会议信息
- 报名字段配置
- 报名提交值
- URL 参数
- 查询参数

## 敏感配置

数据库连接、密钥、令牌等敏感配置必须放在环境变量或安全配置系统中。

`.env` 不提交到版本库。

仓库根目录通过 `.gitignore` 忽略 `.env`、`.env.*`、虚拟环境和本地缓存文件。需要提交示例配置时，应使用 `.env.example`。

## 前后端责任

前端负责提升交互体验和基础提示，后端负责最终安全判断和数据一致性。
