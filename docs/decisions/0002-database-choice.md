# 0002 数据库选择

## 状态

已接受。

## 决策

本地早期开发可以使用 SQLite，正式环境选择 PostgreSQL。

MySQL 不作为第一优先选择，但仍然是可接受备选。

## 原因

知会虽然第一版功能较轻，但后续会逐步扩展：

- 多会议管理
- 动态报名字段
- 动态配置
- 报名数据管理
- 页面配置
- 报名审核
- 数据统计
- 权限管理

PostgreSQL 在关系数据、约束、索引、事务和 JSON 数据处理方面都适合长期扩展。对于会议设置、字段选项和未来页面配置等半结构化数据，PostgreSQL 的 JSON 能力也更自然。

SQLite 适合早期本地开发，部署简单，便于快速启动。

## 为什么不优先选择 MySQL

MySQL 是成熟可靠的关系数据库，但本项目预期会同时处理关系数据和部分 JSON 配置数据。综合长期扩展、约束表达和 JSON 使用体验，PostgreSQL 更适合作为正式环境首选。

## MySQL 作为备选

如果部署环境、团队经验或运维条件更偏向 MySQL，MySQL 仍然可以作为备选方案。届时需要重新评估字段类型、JSON 查询、索引策略和迁移脚本。
