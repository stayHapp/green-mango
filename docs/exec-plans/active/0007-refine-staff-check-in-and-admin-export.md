# 优化工作人员签到与管理员导出

## 任务名称

优化工作人员端签到能力，并将签到表导出调整至管理员端。

## 背景

现场工作人员应专注于嘉宾核验和签到操作；全量嘉宾签到数据导出属于会议管理职责，应由管理员完成。本次同时补充工作人员端的嘉宾搜索核验与网络异常提示，提升现场可用性。

## 用户确认状态

用户已确认执行以下范围：管理员端导出签到表；工作人员端搜索核验；工作人员端仅处理网络异常。

## 目标

- 管理员可在会议详情中导出全量嘉宾签到表。
- 工作人员端不再显示或提供签到表导出。
- 工作人员可按姓名、手机号、单位、座位号搜索嘉宾并核验签到状态。
- 工作人员端显示网络状态，并在离线时阻止新的签到操作。

## 不在本次范围内的内容

- 不实现离线签到、待同步队列或网络恢复后的自动补传。
- 不实现异常到场、撤销签到、补签和多签到点。
- 不接入真实后端 API 或修改数据库。

## 技术决策与待确认项

- 保持 JSON mock 数据和浏览器端 xlsx 生成方式不变。
- 使用浏览器 `online` 与 `offline` 事件监测网络状态。
- 搜索仅在当前已授权会议的已加载嘉宾列表中进行。

## 分步计划

1. 将签到表导出逻辑与入口迁移至管理员会议详情页。
2. 为工作人员签到页增加搜索核验和网络异常状态处理，并移除导出功能。
3. 同步产品文档与 MVP 范围。
4. 运行前端构建验证，并记录完成结果。

## 验收标准

- 管理员登录后可导出包含全部嘉宾与签到状态的 xlsx 文件。
- 工作人员页面不显示导出入口。
- 输入姓名、手机号、单位或座位号后，工作人员可看到匹配嘉宾及签到状态。
- 浏览器离线时页面提示网络异常，扫码和手动签到操作不可执行。
- `npm run build` 通过。

## 验证方式

```bash
cd frontend
npm run build
```

并在浏览器中手动检查管理员导出、工作人员搜索和浏览器离线状态提示。

## 风险与注意事项

- 浏览器网络状态只能反映设备连接状态，不能完全证明业务服务器可用；真实 API 接入后仍需处理请求失败。
- 当前导出在浏览器本地生成，导出的内容受已加载 mock 数据限制。
