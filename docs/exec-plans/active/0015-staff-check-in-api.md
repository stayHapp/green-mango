# 工作人员签到 API

## 背景

会议、嘉宾和工作人员授权 API 已完成，当前 MVP 的核心闭环需要工作人员以嘉宾二维码或人工核验方式完成签到。

## 用户确认状态

用户已明确要求继续实现。本任务实现工作人员签到和签到记录查询，不实现摄像头调用或前端联调。

## 目标

- 工作人员可使用嘉宾二维码 token 完成签到。
- 工作人员可通过嘉宾 ID 完成人工签到。
- 工作人员可查看当前会议签到记录。
- 保障会议结束后不能签到、非本会议二维码不能签到、同一嘉宾不能重复签到。

## 不在本次范围内的内容

- 不实现前端摄像头、二维码图片生成、撤销签到、离线签到或导出。
- 不实现按姓名、手机号、单位、座位号搜索嘉宾。

## 技术决策与待确认项

- 使用现有 `X-Staff-Id` 开发期请求头校验工作人员账号与会议授权。
- `POST /scan` 接受二维码 token，`POST /manual` 接受嘉宾 ID；二者共用同一签到服务。
- 签到表唯一约束作为并发下的最终防线；服务层提前返回 409 提示重复签到。

## 分步计划

1. 定义签到请求、响应和业务错误语义。
2. 实现工作人员会议授权、签到创建和签到记录查询服务、路由。
3. 覆盖扫码成功、手动签到、重复签到、过期和跨会议 token 测试。
4. 更新 API 文档并运行完整验证。

## 验收标准

- 仅被授权工作人员可操作会议签到。
- 有效的本会议二维码可签到一次，重复操作返回 409。
- 会议结束后和其他会议嘉宾的二维码均不能签到。
- 签到记录包含嘉宾、工作人员、方式和时间。

## 验证方式

- 在 `backend` 目录使用 Python 3.12 虚拟环境运行 `pytest`。
- 使用临时 SQLite 数据库运行 `alembic upgrade head`。
- 运行 `git diff --check`。
