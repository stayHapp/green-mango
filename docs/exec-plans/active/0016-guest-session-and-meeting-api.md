# 嘉宾登录与嘉宾会议 API

## 背景

管理员录入嘉宾、工作人员签到 API 已完成。当前需要补齐嘉宾端的登录、查看个人会议和展示签到二维码 API，形成三端数据接口闭环。

## 用户确认状态

用户已明确要求继续先实现 API。本任务实现嘉宾登录和嘉宾会议读取，不接入前端真实请求。

## 目标

- 嘉宾可按会议、姓名和手机号登录。
- 嘉宾可获取自己的会议列表、会议详情和二维码 token。
- 嘉宾只能访问自己的会议和二维码。

## 不在本次范围内的内容

- 不实现 JWT、刷新令牌、短信验证或生产会话管理。
- 不实现嘉宾动态字段值展示、会议服务内容或个人资料修改。

## 技术决策与待确认项

- 开发期登录成功后返回嘉宾基本信息；后续请求以 `X-Guest-Id` 请求头表示当前嘉宾。
- `X-Guest-Id` 仅用于本地开发和测试，正式认证完成后必须被安全会话令牌替代。
- 二维码接口仅返回 token 与会议结束时间；实际图像由前端现有二维码库生成，签到有效性仍由工作人员签到 API 最终判断。

## 分步计划

1. 定义嘉宾登录、会话和二维码响应结构。
2. 实现嘉宾身份依赖、登录和会议查询服务、路由。
3. 覆盖登录成功/失败、跨会议访问拒绝和二维码响应测试。
4. 更新文档并运行完整验证。

## 验收标准

- 嘉宾只能以所属会议内的姓名和手机号成功登录。
- 未启用嘉宾不能登录。
- 嘉宾只能查询自己的会议及二维码。
- 二维码响应不包含姓名、手机号等敏感信息。

## 验证方式

- 在 `backend` 目录使用 Python 3.12 虚拟环境运行 `pytest`。
- 运行 `git diff --check`。

## 风险与注意事项

- `X-Guest-Id` 可被伪造，只能用于开发测试，不能用于生产环境。
