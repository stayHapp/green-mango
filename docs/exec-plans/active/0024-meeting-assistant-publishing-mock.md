# 会议助手配置与发布 Mock 前端

## 背景

嘉宾端已经存在会议日程、会议手册、天气情况、路线指引和联系我们五个会议功能入口，但点击后只有统一占位提示。现阶段先建立管理员编辑、发布与嘉宾查看的前端闭环，后端实现和 API 联调后置。

## 本次范围

- 保留五个现有功能名称、说明和嘉宾入口。
- 管理员按会议编辑每项功能的正文、未发布提醒和发布状态。
- 嘉宾点击任一功能后进入对应的独立页面。
- 已发布功能在独立页面中展示管理员正文。
- 未发布功能在独立页面中展示管理员配置的提醒。
- 会议日程使用日期标签、纵向时间线和日程卡片呈现。
- 使用前端 Mock 内存数据验证完整流程，不新增后端接口。

## 不在本次范围

- 不实现五项功能各自的结构化内容模型。
- 不接入天气、地图、文件或其他第三方服务。
- 不修改嘉宾登录、个人信息和签到二维码流程。

## 验证方式

- `cd frontend && npm run build`
- 管理员编辑并发布一项功能，嘉宾端查看发布内容。
- 管理员保持一项功能未发布并修改提醒，嘉宾端查看提醒。
- `git diff --check`

## 完成记录

- 管理员会议详情新增“会议助手”页签，可编辑五项功能的正文、未发布提醒和发布状态。
- 嘉宾端使用会议信息上方的图标展开五个功能入口，点击任一入口后跳转到统一视觉体系的独立详情页。
- 独立详情页使用渐变标题栏和返回图标；会议日程使用日期标签、纵向时间线和日程卡片呈现，其他功能使用内容卡片呈现。
- 未发布功能同样进入独立详情页，但只展示管理员配置的提醒，不泄露草稿正文。
- Mock 配置按会议写入浏览器本地存储，支持管理员与嘉宾跨页面、跨角色和刷新验证。
- 浏览器使用会议 ID `1` 验证“会议日程”跳转至 `/guest/meetings/1/assistant/agenda`，并展示两行测试日程正文。
- 浏览器验证返回图标可回到嘉宾会议页面；“会议手册”跳转至独立页面后只展示未发布提醒。
- 前端生产构建和 `git diff --check` 通过。
