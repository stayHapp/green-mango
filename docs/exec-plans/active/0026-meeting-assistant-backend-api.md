# 会议助手后端配置与 API

## 背景

会议助手五项功能已完成前端 Mock 编辑、发布、查看和撤回验证，产品边界也已确认。下一阶段需要把浏览器本地配置迁移为受会议授权保护的服务端数据，并保证未发布草稿不会返回嘉宾端。

## 本次范围

- 新增会议助手配置模型和 Alembic 迁移。
- 新会议创建时生成五项默认配置，历史会议读取时补齐缺失配置。
- 实现管理员配置列表和单项更新接口。
- 实现嘉宾单项公开内容接口。
- 覆盖管理员授权、固定功能标识、字段长度、默认配置和草稿隔离测试。
- 同步更新数据库与 API 文档中的实现状态。

## 不在本次范围

- 不联调前端，前端从 `localStorage` 迁移作为后续独立小任务。
- 不接入真实天气、地图、文件或消息服务。
- 不实现结构化日程、联系人或路线字段。
- 不实现嘉宾二维码“已签到”视觉标记；该需求已记录在嘉宾流程和 API 后续设计中。

## 设计约束

- 每个会议固定五项功能：`agenda`、`manual`、`weather`、`route`、`contact`。
- 同一会议与功能标识组合唯一。
- 管理员正文最长 20,000 字符，未发布提醒最长 500 字符，均只接受纯文本。
- 未发布嘉宾响应的 `content` 必须为 `null`。
- 天气数据不写入会议助手配置表。

## 实施步骤

1. 新增模型、关系、迁移和默认配置常量。
2. 新增 Pydantic 请求与响应结构。
3. 新增管理员和嘉宾服务及路由。
4. 增加权限、校验、默认初始化和草稿隔离测试。
5. 执行迁移、后端测试和仓库检查并更新完成记录。

## 验证方式

- `cd backend && .venv/bin/alembic upgrade head`
- `cd backend && .venv/bin/pytest`
- `./scripts/check_all.sh`
- `git diff --check`

## 完成记录

- 新增 `meeting_assistant_features` 模型和 `20260716_0005` 迁移，固定功能标识由数据库检查约束保护，同一会议与功能标识组合唯一。
- 新会议在创建事务内生成五项默认配置；历史会议首次读取时自动补齐缺项且不覆盖已有内容。
- 管理员可以读取五项完整配置并更新单项正文、未发布提醒和发布状态。
- 嘉宾只能读取所属会议的单项功能；未发布响应强制返回 `content: null`，已保存草稿不会泄露。
- 新增 4 项会议助手 API 测试，并扩展核心模型表与唯一约束测试。
