# 会议助手前端 API 联调

## 背景

会议助手模型、迁移以及管理员和嘉宾 API 已完成，但管理员编辑和嘉宾查看仍读取浏览器 `localStorage` Mock，无法跨浏览器共享配置，也无法验证服务端草稿隔离。

## 本次范围

- 新增会议助手前端 API 客户端和 snake_case（蛇形命名）字段转换。
- 管理员会议详情改用真实配置列表和更新接口。
- 嘉宾会议助手详情页改用真实公开内容接口。
- 固定五项入口标题与说明保留在前端，不请求配置正文。
- 增加会议助手独立加载、保存和错误提示。
- 删除不再使用的会议助手 `localStorage` Mock 服务。

## 不在本次范围

- 不接入真实天气服务。
- 不实现结构化日程、文件、地图或联系人。
- 不实现二维码“已签到”标记。

## 验证方式

- `cd frontend && npm run build`
- `cd backend && .venv/bin/pytest`
- 管理员发布天气内容，嘉宾查看已发布内容。
- 管理员撤回天气内容，嘉宾只能查看未发布提醒。
- 刷新管理员页面，确认配置仍由后端持久化。
- `git diff --check`

## 完成记录

- 新增 `frontend/src/api/meetingAssistant.ts`，完成管理员列表、单项更新和嘉宾单项读取接口封装。
- 管理员会议详情将会议助手作为独立资源加载，失败时不影响会议、嘉宾和工作人员等其他数据。
- 保存流程补齐异常捕获和加载状态清理。
- 嘉宾详情页校验固定功能标识后读取真实 API，并使用统一后端错误提示。
- 固定入口定义迁移到 API 模块，旧 `localStorage` Mock 文件已删除。
- 浏览器验证管理员发布天气配置后刷新页面仍保持“已发布”，嘉宾端展示后端保存的测试正文。
- 浏览器验证管理员撤回天气配置后，嘉宾端仅展示后端保存的未发布提醒，已保存正文不再出现。
