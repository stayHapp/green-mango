# 管理员签到统计前端 API 联调

## 背景

管理员会议详情中的签到数量、嘉宾签到状态和签到记录仍读取前端 Mock，与工作人员真实签到和嘉宾二维码签到状态不一致。

## 本次范围

- 新增管理员签到统计 API 客户端。
- 管理员会议详情使用真实总数、已签到数、未签到数和签到记录。
- 签到统计作为独立资源加载，失败时不影响会议其他资料。
- 显示嘉宾姓名、手机号、签到方式、工作人员和签到时间。
- 统一修复 SQLite 签到时间的 UTC 响应语义。

## 不在本次范围

- 不修改工作人员签到工作台。
- 不修改签到表 Excel 导出。
- 不实现实时推送或自动轮询。

## 验证方式

- `cd backend && .venv/bin/pytest`
- `cd frontend && npm run build`
- 使用已有真实签到记录验证管理员统计和明细。
- `git diff --check`

## 完成记录

- 新增 `frontend/src/api/adminCheckIns.ts`，完成统计和记录字段转换。
- 管理员会议详情移除签到 Mock，顶部计数、嘉宾签到状态和签到页签均改用真实接口。
- 签到统计作为独立资源加载并提供独立错误提示。
- 服务端管理员签到明细恢复 SQLite 丢失的 UTC 时区，测试断言响应带 UTC 标识。
- 后端 27 项测试和前端生产构建通过。
- 浏览器验证会议 ID `1` 顶部显示“签到 1/1”，嘉宾列表显示“已签到”；签到记录页展示李文博、手机号、现场一组、扫码方式和正确的本地签到时间。
