# 工作人员签到工作台前端 API 联调

## 背景

工作人员登录已使用真实会话，但负责会议、嘉宾搜索、扫码签到、人工签到和签到列表仍读取前端 Mock，无法与管理员统计及嘉宾二维码状态形成真实闭环。

## 本次范围

- 新增工作人员会议与签到 API 客户端。
- 负责会议列表接入真实接口。
- 签到页接入真实会议、嘉宾搜索和签到记录。
- 扫码与人工签到接入真实接口，并处理成功、重复签到和业务错误。
- 保留摄像头扫码、离线禁用和演示 token 填充能力。
- 统一工作人员签到响应的 UTC 时间语义。

## 不在本次范围

- 不删除种子脚本、演示账号、测试数据或样例数据。
- 不实现离线缓存、签到撤销或实时推送。
- 不修改管理员工作人员维护流程。

## 验证方式

- `cd backend && .venv/bin/pytest`
- `cd frontend && npm run build`
- 工作人员登录后看到真实负责会议。
- 使用样例 token 验证扫码成功或重复签到提示。
- 使用未签到样例嘉宾验证人工签到并刷新列表。
- `git diff --check`
