# 嘉宾会议页面前端 API 联调

## 背景

嘉宾登录、个人资料读取、二维码和会议助手已经使用真实接口，但“我的会议”和会议详情仍读取前端 Mock，导致登录后的主要访问路径没有完全闭环。

## 本次范围

- 新增嘉宾会议列表与详情 API 客户端。
- “我的会议”切换为真实接口，并补齐加载和错误状态。
- 会议详情切换为真实接口，并继续复用登录时读取的真实个人资料、二维码和会议助手。
- 使用现有嘉宾样例会话完成浏览器闭环验证。

## 不在本次范围

- 不删除或重置种子脚本、演示账号和测试数据。
- 不新增嘉宾自行修改资料能力；当前后端仅开放个人资料查询。
- 不调整公开报名与审核流程。

## 验证方式

- `cd frontend && npm run build`
- `cd backend && .venv/bin/pytest`
- 嘉宾登录后可看到真实会议并进入详情。
- 详情页展示真实个人资料和已签到二维码状态。
- `git diff --check`

## 完成记录

- 嘉宾会议列表和详情已改用真实 API，并提供加载、空数据和请求失败反馈。
- 登录时读取的真实个人资料、签到二维码状态和会议助手继续在详情页组合展示。
- 浏览器使用现有李文博样例数据验证会议列表、详情资料及“已签到”覆盖标记；未删除或重置任何样例数据。
- 前端生产构建、后端完整测试和差异格式检查均通过。
