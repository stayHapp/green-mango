# 天气与路线指引生产化

## 背景

会议助手天气页当前使用前端测试数据，路线指引仅展示通用正文。需要将天气切换为和风天气真实数据，并为路线增加管理员地图选点、手机导航和到场说明。

## 执行顺序

1. 接入和风天气后端配置、查询、缓存、嘉宾 API 和自动化测试。
2. 嘉宾天气页切换真实数据，并保留未配置、超时和超出预报日期的降级提示。
3. 为会议增加导航名称、地址、高德坐标和到场说明字段及迁移。
4. 管理员会议编辑增加高德地点搜索与选点。
5. 嘉宾路线页增加手机导航按钮和到场说明。

## 当前完成记录

- 和风天气后端代理、缓存、嘉宾接口和真实页面已完成。
- 会议导航名称、地址和高德坐标字段及 Alembic 迁移已完成。
- 管理员地点搜索选择、嘉宾导航按钮与到场说明页面已完成。
- 天气服务优先使用管理员确认坐标，历史会议继续使用地点文字降级匹配。
- 本地环境仍需配置 `AMAP_WEB_SERVICE_KEY` 后才能进行真实高德地点搜索。

## 安全与数据约束

- 和风天气凭据、高德 Key 和安全密钥只从环境变量读取，不提交仓库。
- 外部服务由后端代理或由受域名限制的地图前端 Key 调用。
- 不重置或删除现有会议、嘉宾、签到及样例数据。

## 验证方式

- 后端单元测试覆盖供应商响应转换、权限和降级。
- 前端生产构建通过。
- 浏览器完成管理员配置、嘉宾天气和路线跳转闭环。
- `./scripts/check_all.sh`
- `git diff --check`
