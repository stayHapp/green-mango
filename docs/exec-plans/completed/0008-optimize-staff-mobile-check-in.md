# 优化工作人员端移动签到界面

## 任务名称

将工作人员会议签到页优化为移动端优先的现场操作界面。

## 背景

工作人员通常使用手机在签到台完成核验和签到。当前页面在窄屏下仍以宽表格展示嘉宾与签到记录，统计卡片纵向堆叠，关键操作触达效率不足。

## 用户确认状态

用户已明确要求优化工作人员端的手机界面。

## 执行前必须读取

已读取并遵守：

- `AGENTS.md`
- `docs/product/staff-workflow.md`
- `docs/product/feature-map.md`
- `docs/exec-plans/template.md`
- `docs/exec-plans/completed/0007-refine-staff-check-in-and-admin-export.md`

## 目标

- 手机端以紧凑统计、全宽搜索和易读嘉宾卡片支持现场核验。
- 手机端以卡片展示签到记录，避免横向滚动表格。
- 手机端增大扫码与手动签到操作的触达范围。
- 桌面端继续使用现有表格布局。

## 不在本次范围内的内容

- 不接入摄像头扫码、离线签到或新的后端 API。
- 不改变签到规则、数据字段与权限边界。
- 不重做管理员端和嘉宾端界面。

## 技术决策与待确认项

- 使用 CSS 媒体查询在 `760px` 以下切换列表表现形式。
- 同一份已加载数据同时服务桌面表格与移动卡片，避免新增数据请求。
- 手机端保留电话拨打、签到状态和手动签到操作。

## 注释要求

本次不新增业务函数；若新增函数，必须遵守 `AGENTS.md` 的完整简体中文文档注释规范。

## 涉及文件

- `frontend/src/views/staff/StaffCheckInView.vue`
- `frontend/src/views/staff/StaffMeetingsView.vue`
- `frontend/src/styles.css`
- `docs/product/staff-workflow.md`

## 分步计划

1. 在工作人员页面增加仅移动端显示的嘉宾和签到记录卡片，并将会议列表简化为会议名称与签到入口。
2. 为工作人员页面补充作用域化的移动端样式。
3. 更新工作人员工作流中的移动端使用说明。
4. 运行生产构建与差异检查。

## 验收标准

- 宽度不超过 `760px` 时，不展示工作人员端的宽表格。
- 嘉宾卡片显示姓名、电话、身份、座位和签到状态，未签到嘉宾可直接操作签到。
- 签到记录卡片显示嘉宾、电话、签到时间和方式。
- 搜索输入框与签到操作在手机端易于点击，页面不产生横向溢出。
- `npm run build` 通过。

## 验证方式

```bash
cd frontend
npm run build
```

并使用浏览器设备模拟模式检查 `375px` 宽度下的工作人员签到页。

## 风险与注意事项

- 该原型依赖 Element Plus 组件的类名和 DOM 结构；升级组件版本后需要复查移动端样式。
