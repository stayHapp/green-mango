# 工作人员嘉宾搜索与签到核验 API

## 背景

工作人员已可扫码或人工签到，但尚不能按现场信息搜索嘉宾并核验签到状态。该能力是现场无法扫码时的必要补充。

## 用户确认状态

用户已明确要求继续实现 API。本任务只实现工作人员会议内嘉宾搜索与签到状态查询。

## 目标

- 工作人员可在已授权会议内按姓名、手机号、单位或座位号搜索嘉宾。
- 每条结果包含嘉宾基本信息、签到状态和签到时间。

## 不在本次范围内的内容

- 不实现管理员签到统计、Excel 导出、撤销签到或前端联调。

## 技术决策与待确认项

- 使用 `GET /api/staff/meetings/{meeting_id}/guests?query=`，空 query 返回会议全部嘉宾。
- SQLite 开发期使用 `lower(...) LIKE` 完成大小写无关包含搜索；正式 PostgreSQL 可在后续评估全文检索。

## 涉及文件

- `backend/app/api/`
- `backend/app/schemas/`
- `backend/app/services/`
- `backend/tests/`
- `docs/architecture/api.md`

## 验收标准

- 已授权工作人员可搜索本会议嘉宾，结果包含签到状态。
- 未授权工作人员不能查看嘉宾列表。
- 搜索覆盖姓名、手机号、单位和座位号。

## 验证方式

- 在 `backend` 目录运行 Python 3.12 虚拟环境下的 `pytest`。
- 运行 `git diff --check`。

## 风险与注意事项

- 当前搜索未分页，适用于 MVP 小规模嘉宾名单；大规模名单需后续加分页与索引策略。
