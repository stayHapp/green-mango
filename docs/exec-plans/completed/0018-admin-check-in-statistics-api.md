# 管理员签到统计与明细 API

## 背景

工作人员已能完成签到，管理员端需要查看会议嘉宾总数、已签到数、未签到数及签到明细，才能完成现场管理闭环。

## 用户确认状态

用户已明确要求继续实现。本任务只实现管理员签到统计与明细查询，不实现 Excel 文件导出。

## 目标

- 已授权管理员可查看会议签到统计和明细。
- 明细包含嘉宾、签到时间、方式与执行工作人员。

## 不在本次范围内的内容

- 不实现 Excel 导出、签到撤销、统计图表或前端联调。

## 技术决策与待确认项

- 使用 `GET /api/admin/meetings/{meeting_id}/check-ins` 返回统计对象和明细数组。
- 未签到数由嘉宾总数减去签到记录数计算；数据库唯一约束确保记录数可代表已签到人数。

## 涉及文件

- `backend/app/api/`
- `backend/app/schemas/`
- `backend/app/services/`
- `backend/tests/`
- `docs/architecture/api.md`

## 验收标准

- 已授权管理员可读取准确的嘉宾总数、已签到数和未签到数。
- 未授权管理员无法读取统计。
- 明细含工作人员显示名与签到方式。

## 验证方式

- 在 `backend` 目录运行 Python 3.12 虚拟环境下的 `pytest`。
- 运行 `git diff --check`。

## 风险与注意事项

- 当前明细不分页，适用于 MVP 小规模会议；大规模会议需后续增加分页与索引优化。

- 已覆盖管理员会议授权校验，并由管理员前端统计页面完成真实 API 联调。
