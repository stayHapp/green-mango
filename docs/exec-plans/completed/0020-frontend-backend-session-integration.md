# 前后端登录会话联调

## 背景

后端三端 API 已在 `codex/backend-complete-api` 完成，前端原先通过 `mockApi` 完成登录和本地会话。本阶段先打通真实 API 基础层、三端登录与统一退出，其他业务页面暂不整体替换。

## 目标

- 使用 `VITE_API_BASE_URL` 配置后端地址。
- 建立 Axios 实例和 Bearer 授权配置。
- 保存管理员、工作人员和嘉宾的真实会话信息。
- 管理员、工作人员使用账号密码登录，嘉宾使用会议 ID、姓名和手机号登录。
- 嘉宾登录前从公开会议接口加载入口信息，登录后读取完整个人资料。
- 三端退出时调用服务端撤销接口，并始终清理本地状态。

## 本阶段边界

- 管理员会议、嘉宾管理、工作人员签到等业务页面仍暂用 Mock，后续逐页替换。
- 未调整已完成的页面视觉设计。
- 未引入新的前端框架或状态库。

## 完成内容

1. 新增 `frontend/src/api/client.ts`、`authStorage.ts`、`sessions.ts`。
2. Pinia 改为保存真实 token、主体、角色和过期时间，并过滤旧版或过期存储。
3. 统一登录页接入管理员、工作人员账号密码接口和后端错误提示。
4. 嘉宾入口接入公开会议、嘉宾登录和个人资料接口，二维码使用后端 token。
5. 顶栏退出调用服务端撤销接口，并清理对应客户端本地会话。
6. 浏览器实测发现并修复嘉宾退出后遗留姓名、手机号和丢失会议入口 ID 的问题。
7. 新增显式执行、幂等的本地联调种子脚本和环境变量示例。
