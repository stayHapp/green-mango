# 管理员会议 API 联调

## 背景

三端真实登录与退出完成后，管理员会议页面仍从 `mockApi` 读取数据，真实用户登录会显示空列表。本阶段只打通管理员会议列表、创建、详情和修改，详情页其他资源保持原有 Mock。

## 完成内容

- 新增 `frontend/src/api/adminMeetings.ts`，统一处理 snake_case 与页面驼峰字段、后端空值和管理员 Bearer token。
- 会议列表使用 `GET /api/admin/meetings`，增加加载状态和错误提示。
- 创建会议使用 `POST /api/admin/meetings`，时间按中国时区提交并展示后端校验错误。
- 会议详情使用 `GET /api/admin/meetings/{meeting_id}`，增加骨架屏和错误返回入口。
- 会议编辑使用 `PATCH /api/admin/meetings/{meeting_id}`，保存后同步标题、地点、状态和入口二维码信息。
- 详情页嘉宾、字段、工作人员、签到等资源仍保留 Mock，作为下一阶段联调范围。

## 验证结果

- 后端测试：`23 passed`，仅有一条既有 TestClient 依赖弃用警告。
- 前端类型检查和生产构建通过，仅有既有大包体积提示。
- 浏览器登录管理员后成功读取真实种子会议。
- 浏览器创建会议 ID `2`：`管理员会议联调验证`。
- 浏览器进入详情后将名称改为 `管理员会议联调验证（已修改）`，地点改为 `杭州联调会场 B 厅`，状态改为 `published`。
- 返回会议列表后修改结果与后端一致，浏览器控制台无应用错误。
- `git diff --check` 通过。
