# 管理员嘉宾 API 联调

## 背景

管理员会议列表、创建、详情和编辑已经接入真实后端，但会议详情中的嘉宾列表、新增嘉宾、嘉宾详情、动态字段和二维码生成仍依赖 Mock，无法验证真实嘉宾管理链路。

## 用户确认状态

用户已明确要求“继续联调”，可以实施本计划。

## 目标

- 管理员会议详情通过真实 API 加载嘉宾列表与动态字段配置。
- 新增嘉宾、查看完整嘉宾资料和批量生成嘉宾二维码使用真实 API。
- 保持现有页面布局和二维码渲染方式，并提供加载、成功和错误反馈。
- 使用真实运行中的前后端完成浏览器联调。

## 不在本次范围内的内容

- 嘉宾 Excel 导入与导出。
- 工作人员管理。
- 管理员签到统计与导出。
- 动态字段配置编辑、嘉宾资料修改和停用。

## 技术决策

- 使用独立的 `adminGuests.ts` 集中完成后端 snake_case（蛇形命名）到页面类型的转换。
- 动态字段的“登录验证”状态通过真实登录字段接口计算，MVP 当前固定为姓名与手机号。
- 嘉宾详情读取后端返回的二维码 token，再由前端 `qrcode` 生成二维码图像。
- 工作人员和签到资源暂时继续使用 Mock，不扩大本次改动范围。

## 实际完成内容

- 新增管理员嘉宾 API 客户端，接入嘉宾列表、新增嘉宾、嘉宾详情、动态字段、登录字段与二维码批量生成接口。
- 会议详情页改为加载真实嘉宾和字段数据，新增嘉宾后自动刷新真实列表。
- 嘉宾详情弹窗从后端读取完整资料，并把真实随机 token 渲染为个人签到二维码。
- 二维码批量生成和嘉宾相关操作增加后端错误提示与完整加载状态清理。
- 调整详情加载层挂载位置，消除 Vue 运行时指令警告。
- 更新前端架构文档，明确当前真实 API 与剩余 Mock 的边界。

- 浏览器使用真实管理员账号进入会议 ID `2`，成功读取空嘉宾列表与空动态字段配置。
- 浏览器新增嘉宾“联调嘉宾”（手机号 `13900000088`），真实列表即时显示单位、职务、标签和座位。
- 浏览器从详情接口读取该嘉宾完整资料，成功显示个人签到二维码。
- 浏览器调用二维码批量生成接口，返回“生成 0 个、已有 1 个”。
- 修正加载指令后重新打开详情，浏览器控制台无警告或错误。
- `git diff --check` 通过后完成本计划。
