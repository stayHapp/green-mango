# 嘉宾二维码已签到状态标记

## 背景

工作人员完成签到后，嘉宾端二维码区域仍保持“请出示二维码”的原始状态，嘉宾和现场人员无法从嘉宾页面确认是否已经签到。

## 本次范围

- 二维码接口返回当前嘉宾的签到状态和签到时间。
- 嘉宾二维码组件加载服务端二维码状态。
- 已签到时在二维码上明显展示“已签到”标记，并调整提示文案。
- 页面刷新或重新登录后仍可从后端签到记录恢复状态。
- 覆盖未签到、签到后和跨会议访问测试。

## 不在本次范围

- 不修改二维码 token、有效期或重复签到校验规则。
- 不实现 WebSocket（网页套接字）或轮询实时刷新。
- 不实现撤销签到。

## 验证方式

- `cd backend && .venv/bin/pytest`
- `cd frontend && npm run build`
- 嘉宾登录后显示未签到二维码提示。
- 工作人员完成签到后刷新嘉宾页，二维码显示“已签到”。
- `git diff --check`

## 完成记录

- 二维码接口新增 `is_checked_in` 和 `checked_in_at`，直接读取嘉宾唯一签到记录。
- SQLite 返回无时区签到时间时按 UTC 恢复，避免嘉宾端本地时间偏差。
- 嘉宾二维码组件改为加载真实二维码状态；已签到时覆盖绿色“已签到”标记、签到时间和无需重复扫码提示。
- 后端测试覆盖未签到和签到后响应，完整 27 项测试通过。
- 浏览器先确认未签到提示，再通过真实工作人员扫码接口完成签到；刷新嘉宾页后显示“已签到”和正确的 Asia/Shanghai 本地时间。
