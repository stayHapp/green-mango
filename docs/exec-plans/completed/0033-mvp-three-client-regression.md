# 三端 MVP 完整回归验收

## 背景

管理员、工作人员和嘉宾三端已完成真实 API 联调，前端 Mock 运行时代码也已收尾。需要基于现有样例数据执行一次端到端回归，确认三个客户端读取的是同一份会议、嘉宾和签到状态。

## 本次范围

- 管理员端：登录、会议列表、会议详情、嘉宾、工作人员、签到统计和会议助手入口。
- 工作人员端：登录、负责会议、嘉宾搜索、签到统计、签到记录和重复签到保护。
- 嘉宾端：会议入口、身份登录、会议列表、个人资料、二维码签到状态和会议助手入口。
- 运行前后端自动化检查并记录验收结果。

## 数据保护

- 不执行 `seed_dev`，不重置数据库。
- 不新增、删除或修改现有会议、嘉宾、工作人员及签到记录。
- 不删除前端样例 JSON、后端种子脚本或测试夹具。

## 验证方式

- `./scripts/check_all.sh`
- `cd frontend && npm run build`
- 使用浏览器逐端核对真实页面与跨端签到状态。
- `git diff --check`

## 验收记录

- 管理员端：会议列表、会议详情、嘉宾资料、工作人员授权、签到 `1/1`、扫码记录、导出入口和五项会议助手配置均正常加载。
- 工作人员端：负责会议、嘉宾 `A01` 搜索、签到 `1/1`、扫码记录和重复签到提示均正常；重复操作未新增记录。
- 嘉宾端：会议列表、真实个人资料、二维码“已签到”覆盖标记、签到时间及天气功能未发布提醒均正常。
- 跨端核对发现工作人员签到时间比管理员和嘉宾端早 8 小时；原因是工作人员接口兼容到无时区 UTC 文本时被浏览器按本地时间解释。
- 已在工作人员 API 适配层仅对无时区日期追加 UTC 标识，保留已有 `Z` 或偏移量；复验后三端统一显示 `2026/7/16 04:38`。
- 离线保护的页面提示、按钮禁用和提交前判断均已完成代码核对；当前浏览器控制环境不暴露 `navigator.onLine`，未在本轮重复模拟系统离线事件。
- 全程未执行种子重置，未新增、删除或修改样例业务数据。
