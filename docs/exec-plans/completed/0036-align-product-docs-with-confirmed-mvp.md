# 统一产品文档 MVP 口径

## 任务名称

根据用户确认的产品决策统一 `docs/product` 全部文档。

## 背景

当前基础 MVP 文档、三端工作流和新增嘉宾端产品规范存在版本口径差异。用户已经确认会议服务属于 MVP、工作人员手工签到属于第一版、嘉宾是会议级身份、暂不支持嘉宾端报名，并同意管理员和工作人员使用账号密码登录。

## 用户确认状态

用户已明确确认上述产品决策并同意执行文档修订。

## 执行前必须读取

- `AGENTS.md`
- `README.md`
- `docs/product/` 下全部文档
- 当前 active 执行计划

## 目标

1. 统一 MVP、功能地图、客户端模型和用户角色。
2. 统一管理员、嘉宾、工作人员与签到工作流。
3. 将会议服务明确纳入 MVP。
4. 将扫码和手工签到共同定义为第一版能力。
5. 将嘉宾限定为会议级身份，不描述跨会议账号能力。
6. 删除嘉宾端自主报名与报名审核的当前产品能力描述。
7. 统一管理员、工作人员账号密码登录与嘉宾姓名手机号核验。

## 不在本次范围内的内容

- 不修改前后端代码、接口、数据库模型或迁移。
- 不删除当前后端可能保留的历史报名接口。
- 不实现嘉宾端 UI 优化。
- 不新增跨会议嘉宾账号体系。

## 技术决策与待确认项

- 管理员和工作人员使用账号密码登录。
- 嘉宾从会议专属入口使用姓名和手机号核验会议级身份。
- “手工签到”作为产品统一名称，内部接口可继续使用 `manual`。
- 当前无待确认项。

## 注释要求

本任务只修改 Markdown 文档，不涉及代码注释。

## 验收标准

1. 五项会议服务在 MVP、功能地图和三端职责中表述一致。
2. 扫码签到与手工签到均属于第一版，撤销签到仍属于后续能力。
3. 嘉宾只访问登录入口所属会议，不再出现跨会议“我的会议”能力。
4. 产品文档不再把嘉宾端报名或报名审核列为当前能力。
5. 登录方式在客户端模型和工作流中一致。
6. `git diff --check` 和文档相对链接检查通过。

## 验证方式

- 使用 `rg` 检查报名、跨会议、手工签到、会议服务和登录方式关键词。

- 运行 `git diff --check`。
