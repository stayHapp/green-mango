# 嘉宾端当前会议首页改版

## 任务名称

按嘉宾端产品规范重构当前会议首页与会议服务入口。

## 背景

嘉宾登录页、会议列表页和会议详情页当前存在重复首页、跨会议列表和隐藏式会议功能入口，与已确认的会议级嘉宾身份和 `guest-experience.md` 不一致。需要在不修改 API 和数据库的前提下，统一嘉宾登录后的落点与当前会议首页。

## 用户确认状态

用户已明确要求开始修改，并已先提交产品文档基线。

## 执行前必须读取

- `AGENTS.md`
- `docs/product/guest-experience.md`
- `docs/product/guest-workflow.md`
- `docs/product/meeting-assistant.md`
- `docs/product/mvp-scope.md`
- `docs/architecture/frontend.md`
- 嘉宾端现有页面、组件、API、路由和全局样式

## 目标

1. 嘉宾登录成功后进入唯一的当前会议首页。
2. 旧的嘉宾会议列表路由按会议级身份自动转到当前会议。
3. 首页按会议概要、参会身份、签到状态、会议服务的顺序展示。
4. 手机号默认脱敏，空信息不占独立行。
5. 五项会议服务直接展示，不再隐藏在抽屉或顶部展开按钮中。
6. 产品文案统一为会议资料、天气提醒、路线导航和联系会务。
7. 已签到状态继续由服务端恢复，并在视觉上高于二维码扫码指引。

## 不在本次范围内的内容

- 不修改后端 API、数据库或迁移。
- 不实现会议服务详情页的完整状态重构。
- 不删除后端会议列表接口，只停止在嘉宾页面展示跨会议列表。
- 不实现实时签到推送。
- 不实现嘉宾自主报名。

## 技术决策与待确认项

- `/guest/login?meetingId=...` 只负责会议入口展示与身份核验；登录成功后跳转 `/guest/meetings/{meetingId}`。
- `/guest/meetings` 保留兼容路由，但登录后直接跳转到会话所属会议。
- 当前会议详情页承载唯一嘉宾首页。
- 内部 `manual` 等功能标识保持不变，只调整展示文案。
- 当前无待确认项。

## 分步计划

1. 收敛登录成功和旧列表路由到当前会议详情。
2. 重构当前会议首页结构及明确退出操作。
3. 调整身份字段、手机号脱敏和会议服务入口。
4. 优化二维码已签到与未签到状态视觉。
5. 同步前端架构文档并运行构建检查。

## 验收标准

1. 嘉宾登录后不再停留在登录页展示第二套首页。
2. 嘉宾不会看到跨会议表格或“我的会议”标题。
3. 当前会议首页直接展示五项会议服务。
4. 页面不显示完整手机号，空单位、职务、座位不显示占位行。
5. 退出登录按钮有明确文字标签。
6. 会议资料等产品文案与产品规范一致。
7. 前端生产构建和 `git diff --check` 通过。

## 验证方式

- `cd frontend && npm run build`

- `git diff --check`
- 条件允许时进行浏览器移动端页面检查。

## 风险与注意事项

- 现有嘉宾会话数据仍包含完整手机号，仅在渲染层脱敏；后续如需更强隐私保护，应调整 API 返回策略。
- 旧 `/guest/meetings` 路由可能被历史导航引用，因此本次保留兼容跳转，不直接删除。
- 两个用户提供的压缩包保持未跟踪状态，不纳入修改。
