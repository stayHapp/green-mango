# 简化嘉宾端首页首屏

## 任务名称

压缩嘉宾端首页信息密度并优化会议服务横向滑动入口。

## 背景

当前嘉宾首页虽然层级完整，但会议概要、身份、签到分别使用大卡片，导致移动端首屏无法同时识别嘉宾身份和签到结果。会议服务使用两列说明卡，也占用较多纵向空间。

## 用户确认状态

用户已明确要求简化首页、优化首屏，并将会议服务调整为滑动功能图标。

## 目标

1. 首屏只保留会议识别、嘉宾身份、签到和会议服务所需信息。
2. 压缩会议概要，去除发布状态、重复栏目标题和低价值说明。
3. 合并身份与签到为参会凭证，优先突出姓名、身份、座位和签到状态。
4. 未签到时保留可扫描二维码；已签到时收起二维码，展示成功状态与签到时间。
5. 会议服务改为可横向滑动的图标入口，保持图标与文字标签同时出现。
6. 同步嘉宾端产品规范与前端架构说明。

## 不在本次范围内的内容

- 不修改后端 API、数据库模型或迁移。
- 不改变签到业务规则和二维码服务端校验。
- 不调整会议服务详情页的数据结构。
- 不增加底部导航或新的会议服务功能。

## 分步计划

1. 重组会议概要与参会凭证结构。
2. 为签到二维码增加首页紧凑展示模式。
3. 将会议服务调整为横向滑动图标入口。
4. 同步产品和前端架构文档。
5. 执行构建检查，并使用移动端视口验证首屏和滑动行为。

## 验收标准

1. 常见移动端首屏可以同时看到会议名称、嘉宾姓名、身份、座位、签到状态和会议服务入口。
2. 已签到首页不再继续展示大二维码。
3. 未签到二维码仍可正常出示给工作人员扫码。
4. 会议服务为单行横向滑动入口，点击区域清晰，图标均有文字标签。
5. 页面不存在发布状态、完整手机号和重复栏目说明等低价值首屏内容。
6. 前端生产构建与 `git diff --check` 通过。

## 验证方式

- `cd frontend && npm run build`
- `git diff --check`
- 使用 390×844 移动端视口检查首屏内容。
- 验证会议服务容器可以横向滚动，入口仍能进入对应详情页。

## 风险与注意事项

- 多行会议名称和长单位名称必须截断或自然换行，不能挤压签到区域。
- 横向滑动不能依赖图标含义，必须保留文字标签和可访问名称。
- 已签到收起二维码后仍需显示明确成功状态和签到时间。

## 完成记录

已于 2026-07-17 完成：

- 会议概要压缩为会议名称、时间和地点，移除首页发布状态和重复栏目标签。
