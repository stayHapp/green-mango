# 优化嘉宾登录页面

## 任务目标

统一嘉宾登录页与当前会议首页的视觉和时间格式，移除生产界面的示例身份入口，并完善字段级校验和手机号用途说明。

## 实施范围

- 调整 `GuestLoginView.vue` 的会议概要、表单校验和提示结构。
- 调整登录页专用样式，不影响管理员端、工作人员端和嘉宾首页。
- 同步嘉宾端产品规范与前端架构说明。
- 不修改后端 API、数据库或身份匹配规则。

## 验收标准

1. 登录页不显示“填入示例信息”。
2. 页面使用浅色背景、蓝色会议卡和白色核验卡，与嘉宾首页一致。
3. 会议时间使用正式中文范围格式，避免中间不自然换行。
4. 姓名和手机号空值分别显示字段级错误，并聚焦第一个错误字段。
5. 身份不匹配等服务端错误显示在提交按钮上方。
6. 手机号输入框下展示简短用途说明。
7. 移动端首屏、生产构建和差异检查通过。

## 验证方式

- `cd frontend && npm run build`
- `git diff --check`
- 使用 390×844 视口检查布局。
- 在不提交身份核验请求的前提下检查空表单校验、错误位置和输入框聚焦。

- 会议时间统一为正式中文范围格式。
