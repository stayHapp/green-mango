# 统一嘉宾登录页与首页会议概要

## 任务目标

复用同一个会议概要组件，使嘉宾登录页与当前会议首页的会议信息内容、格式和样式保持一致；同时精简手机号说明，并将身份核验文案调整为报名信息来源。

## 实施范围

- 抽取嘉宾端共享会议概要组件。
- 登录页与当前会议首页复用共享组件。
- 删除手机号用途说明。
- 将身份核验说明和空字段错误统一为“报名时填写”的姓名与手机号。
- 同步产品规范和前端架构说明。

## 不在本次范围

- 不增加嘉宾端报名功能。
- 不修改后端 API、数据库、路由或身份匹配规则。
- 不调整管理员端和工作人员端。

## 验收标准

1. 登录页和首页会议概要使用同一组件。
2. 两个页面的会议名称、时间、地点、图标和时间格式完全一致。
3. 登录页不显示手机号用途说明。
4. 核验说明与字段错误使用“报名时填写”的口径。
5. 生产构建、差异检查和移动端检查通过。

## 验证方式

- `cd frontend && npm run build`
- `git diff --check`
- 使用 390×844 视口检查登录页和当前会议首页。

## 完成记录

- 新增 `GuestMeetingSummary.vue`，统一会议名称、时间、地点、图标和视觉样式。
- 嘉宾登录页与会议首页已复用该组件，避免相同信息在两个页面发生样式漂移。
- 删除手机号用途说明，将身份核验说明和必填错误统一为“报名时填写”的口径。
- 同步更新嘉宾端产品规范与前端架构说明。
- `npm run build`、`git diff --check` 和 390×844 移动端浏览器检查均通过；浏览器控制台无错误。
