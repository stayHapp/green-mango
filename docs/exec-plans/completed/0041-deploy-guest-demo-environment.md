# 嘉宾端测试环境部署

## 目标

将当前 Vue 前端与 FastAPI 后端部署到用户指定的测试服务器，通过 `cssat.wenguang.top` 提供可供领导查看的嘉宾登录和会议首页，并使用不含真实个人信息的测试数据。

## 范围

- 使用 Nginx 提供前端静态资源、SPA 路由回退和 `/api` 反向代理。
- 使用独立 Python 虚拟环境运行 FastAPI，由 systemd 管理进程。
- 测试环境暂用服务器本地 SQLite，不接入正式数据库。
- 执行 Alembic 迁移并创建可重复使用的演示会议与嘉宾。
- 配置域名与 HTTPS；证书签发受 DNS、生效时间和服务器环境约束。
- 不部署生产凭据，不导入真实嘉宾数据，不修改本地开发数据库。

## 执行步骤

1. 只读检查服务器系统、端口、Nginx、Python、磁盘空间及域名解析。
2. 在本地运行后端测试和前端生产构建。
3. 准备测试环境部署配置，上传经过验证的源码或构建产物。
4. 在服务器创建独立目录、虚拟环境、SQLite 数据库和 systemd 服务。
5. 配置 Nginx、SPA 路由回退、API 代理及 HTTPS。
6. 执行迁移和演示数据初始化。
7. 从公网验证健康检查、嘉宾登录、会议首页和关键静态路由。
8. 更新部署文档并记录最终访问方式和验证结果。

## 验证标准

- `https://cssat.wenguang.top/api/health` 返回成功。
- 会议入口在手机和桌面浏览器均可打开。
- 测试嘉宾可以使用姓名和手机号登录。
- 嘉宾首页可以读取会议、个人资料、二维码和会议服务状态。
- 页面刷新或直接访问前端子路由不会返回 Nginx 404。
- 后端服务重启后能够自动恢复。

## 安全说明

- 用户提供的服务器密码不写入仓库、脚本、环境文件或命令历史。
- 部署完成后建议立即轮换已在聊天中暴露的服务器密码，并改用 SSH Key（安全外壳密钥）登录。
- 测试账号仅用于演示，后续正式试用前必须更换。
