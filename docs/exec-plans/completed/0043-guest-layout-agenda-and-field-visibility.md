# 嘉宾端视口、日程样式与字段呈现配置

## 任务名称

统一嘉宾端页面视口、修复会议日程样式并完善嘉宾字段呈现配置。

## 背景

当前嘉宾登录页使用 420 像素手机视口容器，但会议首页使用 720 像素、会议服务页使用 920 像素内容宽度，桌面端产生撑满和拉伸。会议日程字号偏大，时间轴节点在部分尺寸下呈现不规则。管理员字段页只维护扩展字段，未包含姓名、手机号、单位、职务、身份和座位等固定嘉宾信息，也不能统一决定嘉宾端呈现字段。

## 用户确认状态

用户已明确提出以上三项调整，可以修改前后端、会议级配置、自动化测试和文档。

## 目标

- 嘉宾登录、会议首页和五项会议服务页在桌面端均使用一致的手机视口宽度。
- 会议日程字号适合手机阅读，时间轴节点稳定呈现为正圆。
- 管理员字段页统一列出固定字段与扩展字段，并可选择嘉宾端呈现字段。
- 嘉宾端只渲染管理员选择且有值的个人字段，登录仍固定使用姓名和手机号。

## 不在本次范围内的内容

- 自定义嘉宾登录字段组合。
- 拖拽排序、复杂字段类型、选项字段和必填规则编辑。
- 删除固定字段或修改固定字段 key。
- 新增主要框架或独立页面装修系统。

## 技术决策与待确认项

- 嘉宾端统一采用登录页现有的 `420px` 最大宽度。
- 固定字段呈现配置保存在 `meeting_settings.settings_json.guest_visible_fields`，不新增数据库列和迁移。
- 动态字段定义继续使用 `guest_fields`；呈现列表允许固定字段 key 与当前会议动态字段 key。
- 未配置时默认展示姓名、手机号、单位、职务、身份、座位及所有 `visible_to_guest=true` 的动态字段，兼容历史会议。

## 分步计划

1. 将嘉宾首页和服务页容器统一为登录页手机视口宽度。
2. 收紧日程日期、时间、标题和卡片尺寸，强制时间轴节点为正圆。
3. 增加会议级嘉宾呈现字段 GET/PUT API，并兼容历史默认值。
4. 管理员字段页合并固定字段和扩展字段，保存字段定义及呈现选择。
5. 嘉宾资料接口返回呈现字段，首页按配置渲染固定与动态资料。
6. 更新产品、数据库和 API 文档，补齐后端测试。
7. 运行全量测试、前端构建和桌面/手机视口浏览器复验。

## 验收标准

- 桌面浏览器中的嘉宾首页和功能页不再横向拉伸，宽度与登录页一致。
- 日程标题、时间和卡片在手机视口内层级协调，节点为正圆。
- 管理员可看到所有固定字段与扩展字段，并保存嘉宾端呈现开关。
- 嘉宾重新登录后只看到已选择且有值的字段。
- 姓名和手机号登录不受呈现开关影响。
- 后端测试、前端生产构建和 `git diff --check` 通过。

## 验证方式

- `./scripts/check_all.sh`
- `cd frontend && npm run build`
- `git diff --check`
- 浏览器验证管理员字段页、嘉宾登录页、首页和会议日程页。

## 风险与注意事项

- 隐藏字段只影响嘉宾端呈现，不影响管理员管理、工作人员核验、签到导出或登录匹配。
- 会议级 JSON 配置更新必须保留其他既有设置，不能整体覆盖 `settings_json`。
- 动态字段已有值时仍受现有全量替换保护；仅修改呈现开关不应触发动态字段替换。
