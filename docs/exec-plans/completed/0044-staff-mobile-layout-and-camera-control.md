# 工作人员端手机布局与摄像头关闭控制

## 背景

工作人员端当前沿用桌面端 1120 像素页面和全宽导航，在电脑端呈现为大面积拉伸；签到页模块间距较大。扫码关闭按钮位于摄像头画面之后，手机端可能被推出首屏，且摄像头启动中的关闭操作缺少可靠的取消处理。

## 目标

- 工作人员会议列表和签到工作台统一使用嘉宾端同款最大 `420px` 手机视口。
- 收紧标题、统计、签到卡片和嘉宾列表的垂直间距，使首要操作整体上移。
- 在摄像头画面上方提供始终可见的“关闭摄像头”按钮。
- 处理摄像头启动、识别、主动关闭和页面卸载的完整生命周期，确保媒体流释放。

## 不在本次范围

- 更换二维码识别库。
- 修改签到接口或数据库结构。
- 重做管理员端样式。

## 实施步骤

1. 为工作人员路由增加独立的手机端应用外壳样式。
2. 调整会议列表与签到页的卡片、间距和字号。
3. 将关闭按钮移动到扫码画面上方，增加启动防重入和取消标识。
4. 在停止、扫码成功、异常和卸载时停止并清理扫码实例。
5. 执行前端构建、项目测试与浏览器视觉验收。

## 验收标准

- 宽屏中工作人员端保持 420 像素居中，不横向拉伸。
- 签到页面首屏能看到会议信息、统计和扫码入口，模块间距紧凑。
- 摄像头打开后无需滚动画面即可看到关闭按钮。
- 点击关闭后扫码状态结束，重复打开仍可正常工作。
- 前端生产构建、项目检查和 `git diff --check` 通过。

## 完成记录

- 工作人员会议列表和签到工作台已统一为最大 `420px` 居中手机视口，并采用与嘉宾端一致的蓝色、圆角卡片风格。
- 会议信息、统计卡、扫码卡、签到结果和嘉宾列表间距已收紧；签到结果空状态缩小后，参会人员模块进入首屏。
- 摄像头画面上方新增“关闭摄像头”按钮，手机扫码启动时禁止重复启动。
- 扫码生命周期增加代次取消、停止和清理处理，主动关闭、识别成功、启动异常与页面卸载均会释放扫描器。
- 浏览器实际开启摄像头后验证关闭按钮可见；连续执行两次开启和关闭均成功，页面日志无运行错误。
- `frontend/npm run build` 通过；`./scripts/check_all.sh` 的 33 项后端测试通过；`git diff --check` 通过。
