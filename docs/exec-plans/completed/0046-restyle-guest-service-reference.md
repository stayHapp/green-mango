# 执行计划：参照嘉宾服务系统重塑嘉宾端视觉

## 任务名称

参照领导提供的 Kimi 嘉宾服务系统，制作「知会」嘉宾端视觉样稿。

## 背景

现有嘉宾端已具备会议入口、姓名与手机号核验、个人参会信息、签到二维码和五项会议服务，但视觉以蓝色渐变和传统 Element Plus 卡片为主。用户希望先看到一版参照系统风格的可运行样稿，用于确认后续设计方向。

## 用户确认状态

用户已明确要求“调出一版我看看”，允许本计划修改前端样式和嘉宾端页面结构。

## 执行前必须读取

- `AGENTS.md`
- `docs/product/guest-experience.md`
- `docs/product/guest-workflow.md`
- `docs/architecture/frontend.md`
- 当前计划

## 目标

- 将会议入口、嘉宾登录和登录后会议首页统一为墨绿色、米白背景、大圆角和轻阴影的移动端视觉。
- 保留现有 Vue、Element Plus、路由、API、登录、二维码和会议服务逻辑。
- 让会议标题、时间地点、参会操作、嘉宾身份、签到凭证和会议服务形成清晰的信息层级。
- 在移动端和桌面端均保持可读、无横向溢出，并可完成核心跳转与登录流程。

## 不在本次范围内的内容

- 不修改数据库结构、后端 API 或权限规则。
- 不改造管理员端和工作人员端。
- 不新增深色主题、动态模块配置或新的会议业务能力。
- 不复制参考系统源码，仅借鉴视觉语言和交互层级。

## 技术决策与待确认项

- 使用现有 Vue 3、Element Plus 与图标组件，不引入新的主要框架或依赖。
- 通过共享 CSS 设计令牌统一嘉宾端三类页面。
- 会议服务继续使用「知会」现有五项固定服务，不改变产品范围。
- 当前无阻塞性待确认项，先交付一版可运行样稿供用户评审。

## 注释要求

遵守 `AGENTS.md`：所有代码注释使用简体中文，每个函数保留或补齐完整中文文档注释，关键逻辑添加中文说明。

## 分步计划

1. 生成完整移动端视觉概念并提取颜色、排版、间距、圆角和组件规则。
2. 调整会议概要、入口页、登录页、个人入场码首页及服务入口结构。
3. 使用共享样式令牌实现视觉系统与响应式布局。
4. 运行前端构建与项目检查。
5. 启动本地服务，使用浏览器检查入口、登录和嘉宾首页，并对照概念图修正视觉差异。

## 验收标准

- 三类嘉宾页面具有一致的墨绿色视觉语言。
- 入口页能展示会议概要、参会登录和报名操作。
- 登录页字段校验与成功登录逻辑保持正常。
- 嘉宾首页能展示个人资料、二维码状态和五项会议服务。
- 360px 及以上移动端无横向溢出，桌面端保持居中的移动端应用外观。
- 前端构建及项目检查通过。

## 验证方式

- `npm run build`
- `./scripts/check_all.sh`
- 浏览器检查桌面视口与移动视口。
- 对照视觉概念图和最终页面截图检查文案、层级、配色、圆角、间距、图标和响应式表现。

## 风险与注意事项

- 现有嘉宾样式集中在全局 CSS，修改时必须限定选择器，避免影响管理员端和工作人员端。
- Element Plus 内部结构需要使用页面级作用域覆盖，避免通用组件样式泄漏。
- 本次样稿保持现有固定服务模型，因此不会完全复制参考系统的动态模块排序。
