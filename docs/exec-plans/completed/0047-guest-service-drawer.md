# 嘉宾端会议服务抽屉

## 背景

嘉宾端当前会议首页直接展示五项会议服务，占用主页纵向空间。用户要求参考已提供的界面，将会议服务收纳到左上角菜单按钮中，并通过左侧抽屉呈现。

## 用户确认状态

用户已明确要求开始修改，并提供抽屉展开状态参考图。

## 执行前必须读取

- `AGENTS.md`
- `docs/product/guest-experience.md`
- `docs/architecture/frontend.md`
- 当前执行计划

## 目标

- 在嘉宾当前会议首页左上角增加会议服务菜单按钮。
- 从首页正文移除常驻会议服务列表。
- 点击菜单后从左侧打开带遮罩的会议服务抽屉。
- 抽屉复用现有五项会议服务及其详情路由，并提供关闭和返回活动首页操作。

## 不在本次范围内的内容

- 不修改会议服务 API、数据结构或发布规则。
- 不调整五项服务详情页内容。
- 不增加新的服务类型。

## 技术决策与待确认项

- 使用现有 Element Plus Drawer（抽屉）组件，方向为从左向右。
- 复用 `MeetingAssistantShortcutGrid`，增加抽屉展示变体，不复制路由逻辑。
- 无待确认项。

## 注释要求

新增函数使用完整简体中文文档注释，关键状态切换补充中文说明。

## 分步计划

1. 增加首页菜单按钮与左侧抽屉结构。
2. 为会议服务快捷组件增加抽屉变体并完成视觉样式。
3. 更新产品与前端架构文档。
4. 执行前端构建，并在浏览器验证打开、关闭、返回及服务跳转。

## 验收标准

- 首页默认不再展示会议服务列表。
- 菜单按钮可访问且点击后抽屉从左侧打开。
- 抽屉包含会议名称、会议服务标题、五项服务、关闭按钮和返回活动首页入口。
- 点击服务后进入对应详情页；按关闭按钮或遮罩可收起抽屉。
- 手机宽度下无横向溢出，键盘焦点样式清晰。

## 验证方式

- `npm run build`
- `git diff --check`
- 内置浏览器当前视口与 390×844 视口交互验证
- 将参考图与浏览器截图通过 `view_image` 对照检查

## 风险与注意事项

- 抽屉通过 Teleport（传送）挂载到页面外层，样式必须使用唯一全局类名，避免作用域样式失效。
- 服务组件原有首页样式较多，抽屉变体样式需要放在全局样式末尾确保覆盖稳定。
