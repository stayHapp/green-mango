# 嘉宾退出入口与会议服务主题统一

## 背景

嘉宾首页退出登录仍位于内容末尾，用户要求将其放到右上角并使用清晰图标。会议服务详情页仍沿用早期蓝紫色视觉，需统一到当前墨绿色主题，其中会议日程按参考图改为更简洁的纵向卡片列表。

## 用户确认状态

用户已明确要求开始修改，并提供会议日程参考图。

## 执行前必须读取

- `AGENTS.md`
- `docs/product/guest-experience.md`
- `docs/product/meeting-assistant.md`
- `docs/architecture/frontend.md`
- 当前执行计划

## 目标

- 嘉宾首页右上角显示带退出图标和“退出”文字的操作。
- 五项会议服务详情页使用与嘉宾首页一致的墨绿色、纯白面板与浅色边框。
- 会议日程使用简洁列表卡片，突出时间、标题以及可选讲者和地点。
- 保持现有纯文本日程数据和五项服务路由兼容。

## 不在本次范围内的内容

- 不修改服务端日程数据结构。
- 不新增日程筛选、提醒或个人收藏。
- 不修改退出登录业务逻辑。

## 技术决策与待确认项

- 使用 Element Plus `SwitchButton` 作为退出图标。
- 日程正文继续逐行解析，并可使用中文竖线 `｜` 或英文竖线 `|` 表达“标题、讲者、地点”。
- 日期行作为轻量分组标题，无法识别的行仍作为普通日程内容展示。
- 无待确认项。

## 注释要求

新增和修改函数必须保留完整简体中文文档注释；解析规则增加关键中文注释。

## 分步计划

1. 调整嘉宾首页顶部工具栏与退出入口。
2. 重构会议服务详情页通用标题区和日程卡片结构。
3. 在全局样式末尾增加墨绿色主题覆盖及响应式规则。
4. 更新产品与前端架构文档。
5. 构建并通过浏览器验证主页、日程和至少一项其他服务详情。

## 验收标准

- 首页顶部左侧为会议服务菜单，右侧为“退出”按钮。
- 页面末尾不再重复显示退出登录。
- 日程详情页视觉结构与参考图一致，时间和标题层级清晰。
- 五项服务详情页不再出现蓝紫渐变主题。
- 手机尺寸无横向溢出，返回和退出操作可访问。

## 验证方式

- `npm run build`
- `git diff --check`
- 内置浏览器当前视口与 390×844 视口验证
- 参考图与浏览器截图使用 `view_image` 对照

## 风险与注意事项

- 本地会议日程可能未发布，视觉验证优先使用只读现有数据；不为截图修改业务数据。
- 旧样式文件存在多轮历史规则，主题覆盖放在文件末尾并使用页面级作用域选择器。
