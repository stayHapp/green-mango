# 简化管理后台界面与操作流程

## 背景

当前管理后台多个页面存在外层页签容器、内容面板、筛选面板和表格面板连续嵌套的问题，边框、圆角和阴影层级过多。会议信息表单纵向分散，主要字段不能在常用桌面视口内快速完成编辑；嘉宾 Excel 导入还会切换到隐藏的独立区块，打断当前管理上下文。

## 用户确认状态

用户已明确要求开始优化，并指定减少嵌套框、压缩会议信息页面、将 Excel 导入改为弹窗。

## 执行前必须读取

- `AGENTS.md`
- `docs/product/admin-workflow.md`
- `docs/architecture/frontend.md`
- `docs/exec-plans/template.md`

## 目标

- 减少管理后台不必要的边框、圆角、阴影和嵌套容器。
- 将会议名称、会议地点和发布控制放在同一行。
- 让会议信息主要编辑内容在常见桌面视口内一屏展示。
- 点击“导入 Excel”后在嘉宾管理当前页面打开弹窗，不再切换页面区块。

## 不在本次范围内的内容

- 不改变后端 API、数据库结构或会议状态业务规则。
- 不重做嘉宾端和工作人员端视觉。
- 不新增第三方组件库或主要基础设施。

## 技术决策与待确认项

- 发布控制放在基础信息第一行最右侧，使用“发布会议 / 撤回为草稿”按钮修改表单草稿，仍由统一保存按钮提交。
- 已结束会议不提供发布切换，只展示“已结束”状态。
- Excel 导入复用现有模板下载、文件选择和导入结果逻辑，只改变承载方式。
- 视觉简化优先采用开放布局、背景分区和细分隔线，保留必要的表格、输入框边界。

## 注释要求

所有新增函数使用完整简体中文文档注释，说明功能、入参、返回值和异常场景；关键状态判断补充中文注释。

## 分步计划

1. 调整会议信息模板，建立紧凑的三段式编辑布局与发布按钮。
2. 删除导入独立区块和路由状态，增加嘉宾页导入弹窗。
3. 调整后台通用样式，移除重复容器边框并优化统计、筛选和表格层级。
4. 更新产品与前端架构文档。
5. 运行前端构建，并在浏览器验证桌面视口、窄屏视口及导入弹窗交互。

## 验收标准

- 会议信息的会议名称、会议地点和发布控制同排展示。
- 常见桌面视口打开会议信息页时，主要编辑表单无需页面纵向滚动。
- 嘉宾管理点击“导入 Excel”后 URL 和当前区块不变化，弹窗正常出现。
- 管理后台主要页面不再出现外框包内框的重复视觉层级。
- 前端构建通过，浏览器控制台没有新增错误。

## 验证方式

- 执行 `npm run build`。
- 使用内置浏览器验证会议信息页首屏高度、嘉宾导入弹窗、页面身份、控制台和窄屏布局。

## 风险与注意事项

- 当前工作区存在其他未提交修改，必须只做小范围补丁，避免覆盖既有工作。
- 文件选择器属于系统界面，浏览器自动化只验证弹窗和触发入口，不上传真实业务文件。
