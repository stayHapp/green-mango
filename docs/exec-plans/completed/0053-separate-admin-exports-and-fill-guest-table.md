# 分离管理员导出职责并铺满嘉宾表格

## 背景

当前嘉宾管理页的“导出”仍下载签到明细，且签到表导出入口出现在数据总览之外的多个页面；嘉宾表格的业务列均为固定宽度，宽屏下无法利用剩余内容空间。

## 用户确认状态

用户已明确要求执行本次调整。

## 目标

- 签到表导出只在数据总览展示。
- 嘉宾管理导出包含嘉宾信息、来源、管理状态和签到状态的嘉宾状态表。
- 嘉宾表格卡片占满内容区，业务列自适应分配剩余宽度，操作列固定在右侧。

## 不在本次范围内的内容

- 不改变嘉宾导入模板和导入流程。
- 不改变签到表原有字段结构。
- 不增加新的前端或后端框架。

## 技术决策与待确认项

- 新增独立的嘉宾状态表导出接口，避免复用签到明细接口造成语义混淆。
- 嘉宾状态表与管理列表口径一致：正式嘉宾及尚未转化的待审核、已拒绝报名申请；已通过申请以正式嘉宾记录呈现，避免重复。
- 表格业务列改用最小宽度，由 Element Plus 自动分配剩余空间；操作列保持固定宽度并固定在右侧。

## 分步计划

1. 新增嘉宾状态表生成与下载接口，并补充工作簿测试。
2. 分离前端两个导出入口和处理函数。
3. 调整嘉宾表格列宽策略与容器宽度。
4. 运行后端测试、前端构建和浏览器交互验证。

## 验收标准

- 仅数据总览出现“导出签到表”。
- 嘉宾管理出现“导出嘉宾状态表”，下载文件包含嘉宾信息、来源、管理状态和签到状态。
- 宽屏下表格卡片与表格占满内容区域，左右内边距为 24px，操作列固定在右侧。
- 自动化测试、构建和浏览器控制台检查通过。

## 验证方式

- 运行管理员 Excel 相关 pytest。
- 运行前端生产构建和差异格式检查。
- 在桌面宽屏浏览器中检查入口、表格宽度、操作列位置，并实际触发两个下载。

## 风险与注意事项

- 自主报名已通过后同时存在申请和正式嘉宾，导出时必须排除已通过申请，防止重复。
- 下载验证应避免把测试文件写入仓库。

## 完成记录

- 数据总览保留签到表导出；嘉宾管理改为嘉宾状态表导出；其他管理区块不再显示签到表导出。
- 嘉宾表格业务列改用最小宽度自适应扩展，容器和表格宽度为 100%，左右内边距为 24px，操作列固定在右侧。
