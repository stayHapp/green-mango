# 管理员端工作流

## 目标

管理员端用于完成会议组织管理、嘉宾信息维护、工作人员管理和签到情况查看。

## 核心流程

1. 管理员登录后台。
2. 创建会议。
3. 编辑会议基础信息。
4. 配置嘉宾信息字段。
5. 录入或导入嘉宾信息。
6. 确认嘉宾使用姓名 + 手机号登录。
7. 创建工作人员账户。
8. 会议现场查看签到情况。
9. 会后查看签到结果。
10. 管理员可在页面顶部退出登录，退出后返回管理与签到入口。

## 会议管理

管理员可以创建和编辑会议基础信息，包括：

- 会议名称。
- 会议说明。
- 会议地点。
- 开始时间。
- 结束时间。
- 会议状态。

会议结束时间同时影响嘉宾二维码有效期。会议结束后，嘉宾二维码应视为过期。

会议发布后，系统展示不可变的会议 ID 与会议入口 URL。管理员可复制该 URL 或将其用于生成会议二维码；扫描入口后仍需完成身份登录。

会议入口 URL 必须使用嘉宾设备能够访问的本机局域网 IP 或正式域名，不得使用仅当前电脑可访问的 `localhost`。

## 嘉宾信息字段配置

第一版转向嘉宾信息字段配置，而不是报名字段配置。

管理员可以为会议配置嘉宾信息字段，例如：

- 姓名。
- 手机号。
- 单位。
- 职务。
- 嘉宾标签。
- 座位号。
- 证件编号。

第一版固定使用姓名 + 手机号作为嘉宾登录验证方式。后续可允许管理员从已配置字段中自由组合登录验证字段。

## 嘉宾管理

管理员可以提前录入嘉宾信息，也可以通过 Excel 批量导入会议嘉宾名单。

第一版支持：

- 单个新增嘉宾。
- 通过 Excel 批量导入嘉宾。
- 编辑嘉宾信息。
- 删除或停用嘉宾。
- 查看嘉宾签到状态。

第一版的批量导入应提供下载模板、字段校验、错误行提示和导入结果摘要。

## 工作人员管理

工作人员账户由管理员创建。

管理员至少需要设置：

- 工作人员姓名。
- 登录账号。
- 初始密码或登录方式。
- 可负责的会议。

第一版不强制细分工作人员权限，工作人员默认只能执行签到相关操作。

## 签到情况查看

管理员可以查看会议签到情况，包括：

- 嘉宾总数。
- 已签到人数。
- 未签到人数。
- 签到时间。
- 执行签到的工作人员。
- 导出包含全量嘉宾、签到状态、签到时间和签到方式的 Excel 签到表。

## 暂不处理

第一版暂不处理复杂权限、多级组织、多租户、短信通知和复杂统计分析；Excel 嘉宾导入与管理员导出签到表属于本期已支持能力。
