# 签到流程

## 目标

签到流程用于确认嘉宾实际到场，并让管理员看到会议签到情况。

## 核心闭环

```text
管理员录入嘉宾 -> 系统生成嘉宾二维码 -> 嘉宾展示二维码 -> 工作人员扫码 -> 系统完成签到 -> 管理员查看签到情况
```

## 前置条件

签到前需要满足：

- 会议已创建。
- 嘉宾已录入。
- 工作人员账户已创建。
- 工作人员有权限操作该会议。
- 嘉宾能登录嘉宾端并展示二维码。

## 二维码有效期

嘉宾二维码按会议结束日期限制有效期。

规则：

- 当前时间未超过会议结束时间时，二维码可用于签到。
- 当前时间超过会议结束时间后，二维码视为过期。
- 过期二维码不能完成签到。

后续实现时应明确使用服务器时间判断，不能依赖客户端时间。

## 签到次数

签到只签一次。

规则：

- 未签到嘉宾首次扫码后，生成签到记录。
- 已签到嘉宾再次扫码，不生成新记录。
- 重复扫码返回已签到状态，并展示原签到时间。

## 签到记录

签到记录至少需要保存：

- 会议。
- 嘉宾。
- 签到时间。
- 执行签到的工作人员。
- 签到方式，第一版为扫码。

## 异常场景

系统应处理：

- 二维码无效。
- 二维码过期。
- 嘉宾不存在。
- 嘉宾不属于当前会议。
- 工作人员未登录。
- 工作人员无权限。
- 嘉宾已签到。

## 管理员查看

管理员可以查看：

- 总嘉宾数。
- 已签到数。
- 未签到数。
- 嘉宾签到明细。
- 签到时间。
- 签到工作人员。
- 导出全量嘉宾签到表。

## 后续扩展

后续可以支持：

- 多签到点。
- 分会场签到。
- 补签。
- 撤销签到。
- 弱网或离线签到。

第一版暂不实现这些扩展能力。
