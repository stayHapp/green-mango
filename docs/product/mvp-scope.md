# MVP 范围

## 第一版要做什么

第一版 MVP 的目标是验证最核心的会议嘉宾管理与签到闭环：

1. 管理员可以创建会议。
2. 管理员可以查看会议列表和会议详情。
3. 管理员可以修改会议基本信息。
4. 一个会议可以有多个管理员。
5. 管理员可以配置嘉宾信息字段。
6. 管理员可以提前录入嘉宾信息。
7. 管理员可以通过 Excel 批量导入会议嘉宾名单，并获得导入结果与错误提示。
8. 管理员可以配置会议日程、会议资料、天气提醒、路线导航和联系会务五项固定会议服务，并分别控制发布状态。
9. 第一版嘉宾固定从会议专属入口使用姓名 + 手机号核验身份；嘉宾身份只在当前会议内有效。
10. 嘉宾可以登录嘉宾端并查看当前会议、个人参会信息和签到状态。
11. 嘉宾可以查看已发布的五项会议服务；未发布功能只展示提醒，不泄露草稿正文。
12. 嘉宾端展示个人签到二维码。
13. 嘉宾二维码按会议结束时间过期。
14. 管理员可以创建工作人员账户。
15. 管理员和工作人员使用账号密码登录管理与签到入口。
16. 工作人员可以扫码完成嘉宾签到。
17. 工作人员可以搜索嘉宾、核验身份与签到状态，并对未签到嘉宾执行手工签到。
18. 工作人员网络断开时，系统提示网络异常并禁止新的签到操作。
19. 一个嘉宾在一个会议中只签到一次，扫码和手工签到遵守相同的唯一性规则。
20. 管理员可以查看签到情况并导出全量嘉宾签到表。

## 第一版暂时不做什么

- 微信登录。
- 支付。
- 短信验证码。
- 邮件通知。
- 嘉宾端自主报名和报名审核。
- 复杂权限系统。
- 复杂统计分析。
- 文件上传。
- 多租户系统。
- 高级页面装修。
- 多次签到、分会场签到、补签、撤销签到。
- 弱网或离线签到。
- 嘉宾登录字段自由组合。

## MVP 验收标准

1. 可以通过管理员端创建一个会议。
2. 可以为会议配置嘉宾信息字段。
3. 可以录入至少一名嘉宾。
4. 可以通过 Excel 导入嘉宾名单，并识别不合法数据。
5. 可以创建至少一名工作人员。
6. 嘉宾可以使用姓名和手机号登录。
7. 嘉宾可以查看当前会议信息、个人参会信息和签到状态。
8. 管理员可以维护五项固定会议服务，嘉宾只能查看已发布正文或未发布提醒。
9. 嘉宾可以展示个人签到二维码。
10. 工作人员可以扫码或手工完成签到。
11. 重复扫码或重复手工签到不会创建第二条签到记录。
12. 会议结束后二维码不能继续签到。
13. 工作人员可以搜索并核验嘉宾的签到状态。
14. 浏览器离线时，工作人员端不能提交新的签到操作。
15. 管理员可以看到嘉宾签到状态并导出签到表。
16. 数据库结构和 API 设计文档与实现保持同步。
