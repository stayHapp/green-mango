# MVP 范围

## 第一版要做什么

第一版 MVP 的目标是验证最核心的活动组织闭环：

1. 管理员可以创建会议。
2. 管理员可以查看会议列表和会议详情。
3. 管理员可以修改会议基本信息。
4. 每个会议可以配置自己的报名字段。
5. 字段类型支持 `text`、`number`、`select`。
6. 字段支持名称、标识、是否必填、排序和选项配置。
7. 用户可以访问某个会议页面。
8. 系统根据会议字段配置动态生成报名表。
9. 用户可以提交报名信息。
10. 管理员可以查看某个会议的报名记录和字段值。

## 第一版暂时不做什么

- 微信登录
- 支付
- 短信验证码
- 邮件通知
- 复杂权限系统
- 复杂统计分析
- 文件上传
- 报名审核
- Excel 导出
- 多租户系统
- 高级页面装修

## MVP 验收标准

1. 可以通过后台创建一个会议。
2. 可以为该会议配置至少 3 个不同类型的报名字段。
3. 可以在前台会议页面看到动态报名表。
4. 必填字段未填写时无法提交。
5. 提交成功后，后台可以看到报名记录。
6. 后台报名详情可以展示用户提交的动态字段值。
7. 数据库结构和 API 设计文档与实现保持同步。
