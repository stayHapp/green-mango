# 用户角色

第一版按客户端定义三类核心用户：管理员、嘉宾、工作人员。

## 管理员

管理员是会议组织方或管理人员。

管理员可以：

- 创建会议。
- 查看会议列表和会议详情。
- 修改会议基本信息。
- 配置嘉宾信息字段。
- 提前录入嘉宾信息。
- 确认嘉宾使用姓名 + 手机号登录。
- 创建工作人员账户。
- 查看嘉宾列表。
- 查看签到情况。

一个会议可以有多个管理员。第一版可以先不细分管理员权限，后续再扩展会议角色和权限模型。

## 嘉宾

嘉宾是被邀请或已确认参加会议的人。

嘉宾可以：

- 登录嘉宾端。
- 查看自己参加的会议。
- 查看会议相关信息。
- 查看个人参会信息展示。
- 展示个人签到二维码。

嘉宾默认由管理员提前录入。系统可以保留嘉宾报名接口作为补充入口。

第一版嘉宾固定使用姓名 + 手机号登录；后续可允许管理员根据已配置字段自由组合登录验证方式。

## 工作人员

工作人员是现场执行签到的人。

工作人员可以：

- 登录工作人员端。
- 查看自己负责的会议。
- 扫描嘉宾二维码。
- 完成签到。
- 查看扫码结果。

工作人员账户由管理员创建。第一版工作人员只处理签到，不具备会议编辑和嘉宾信息编辑权限。

## 非核心角色

公开访问者和自主报名用户可以作为后续扩展角色处理。当前 MVP 的核心不是公开报名，而是嘉宾信息管理和签到。
